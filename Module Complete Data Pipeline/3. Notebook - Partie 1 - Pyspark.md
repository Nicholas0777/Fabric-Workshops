# Notebook : Premières manipulations
# 1. [Pratique] Importer le fichier Pays.csv dans le notebook avec PySpark
1. Dans le menu **...**, sélectionner pour **Pays.csv**, sélectionnez **Charger les données** > **Spark**. Une nouvelle cellule de code contenant le code suivant devrait être ajoutée au notebook :<br>
    ``` python
    df = spark.read.format("csv").option("header","true").load("Files/Pays.csv")
    # df now is a Spark DataFrame containing CSV data from "Files/Pays.csv".
    display(df)
    ```
    
2. Cliquer sur le bouton **&#9655;** à gauche de la cellule pour l'éxcuter  (**Raccourci :** Vous pouvez également faire **CTRL** + **Entrée**)
3. En dessous de la cellule nouvellement éxécutée vous pourrez voir le résultat d'affichage

    ``` python
    df = spark.read.format("csv").option("header","true").load("Files/Pays.csv", encoding='ISO-8859-1', sep=";")
    # df now is a Spark DataFrame containing CSV data from "Files/Pays.csv".
    display(df)
    ```
    Le format est à présent satisfaisant
    <img src="https://github.com/Nicholas0777/Fabric-Workshops/blob/main/Module%20Complete%20Data%20Pipeline/images/Notebook/Notebook-Pays-csv-good-import.png" alt="" width="80%"/> <br>
5. On vérifie le schéma du dataframe Spark.
    ``` python
    df.printSchema()
    ```
    _Output :_
    ```

    |-- region: string (nullable = true)
    |-- code_pays: string (nullable = true)
    |-- pays: string (nullable = true)
    ```

6. Sauvegarder le dataframe dans le **Lakehouse Silver**
    ``` python
    spark_df = spark_df.toDF( *(column.replace(' ', '_') for column in spark_df.columns) ) # On retire les espaces des noms de colonnes
    spark_df.write.format("delta").mode("overwrite").option('encoding','ISO-8859-1').option("nullValue", "\\N").saveAsTable("DE_LH_Silver.pays")
    # option('encoding','ISO-8859-1') = On precise encore l'encodage d'ecriture des donnees dans le lakehouse
    # option("nullValue", "\\N") = Sert a indiquer que les valeurs nulles sont egale a "\N" et non "NA" (Rappel : Code pays de Namibie = "NA")
    ```
    Une fois le code exécuté, aller dans le **Lakehouse Silver** afin de vérifier que la table a été créée.<br>

Pour créer une nouvelle colonne **URL**, on va se servir de la méthode `withColumn` (elle sert soit à créer une nouvelle colonne, soit à remplacer une colonne existante) ainsi que de fonctions natives à **SQL** via `pyspark.sql.functions` pour réaliser la concatenation de l'ID du site et de la racine de l'URL.

3. Dans la colonne **Catégorie**, selon que le site soit le suffixe *_danger* vient indique que le site est en péril. Pour continuer notre travail, nous allons dans un premier temps créer une nouvelle colonne contenant cette information puis nous allons uniformiser les valeurs la colonne **Catégorie** en faisant un remplacement de valeurs. <br>
Pour ajouter la colonne **En péril** dans le dataframe, qui indique si un site est en péril ou non, nous allons utiliser la méthode `withColumn`:

    ``` python
    df = df.withColumn( "En péril", when(col("Catégorie").contains('danger'),True).otherwise(False) )
    
    # Afficher les 5 premieres lignes du dataframe
    display( df.limit(5) )
    ```

4. Avant d'effectuer le remplacement des valeurs de la colonne **Catégorie** il faudrait avoir une idée du nombre de remplacements à effectuer (cela permet également d'utiliser une méthode de remplacement adéquate). Pour avoir cette liste de valeurs, on utilisera la **méthode** `distinct()`:

6. Nous allons créer un nouveau DataFrame réordonné tout en conservant une copie de l'original afin de pouvoir revenir en arrière si nécessaire. Pour ce faire, nous allons utiliser le code suivant :

    > **Remarques :** <br>
    Bien que Spark accepte les noms de colonnes commençant par une majuscule, il est généralement préférable de suivre une convention de nommage en minuscules avec des underscores pour améliorer la lisibilité et éviter les problèmes potentiels avec d'autres outils ou systèmes.<br>
    Par contre, Spark n'accepte pas de sauvegarder des tables qui ont des noms de colonnes ayant des caractères spéciaux ou des espaces. Il est donc important de s'assurer que les noms de colonnes sont conformes à ces restrictions lors de la manipulation de données avec Spark. <br><br>

7. Avant de sauvegarder le dataframe dans le Lakehouse (comme table delta), on va vérifier que le bon type est associé à chacune des colonnes.
    ``` python
    spark_df = df_reorder
    spark_df.printSchema()
    ```
    _Output :_
    ```
    root
    |-- id: long (nullable = true)
    |-- site: string (nullable = true)
    |-- categorie: string (nullable = true)
    |-- site deliste: boolean (nullable = true)
    |-- en peril: boolean (nullable = false)
    |-- pays: string (nullable = true)
    |-- url: string (nullable = true)
    ```

    Le type de **id** n'est pas le bon, on va alors préciser le bon type avec la ligne de code suivante :
    ```python
    from pyspark.sql.types import IntegerType, StringType, BooleanType
    spark_df = df_reorder.withColumn("id",col("id").cast(IntegerType())) \
                    .withColumn("site",col("site").cast(StringType())) \
                    .withColumn("categorie",col("categorie").cast(StringType())) \
                    .withColumn("site deliste",col("site deliste").cast(BooleanType())) \
                    .withColumn("en peril",col("en peril").cast(BooleanType())) \
                    .withColumn("pays",col("pays").cast(StringType())) \
                    .withColumn("url",col("url").cast(StringType())) \
                
    spark_df.printSchema()
    ```
    _Output :_
    ```
    root
    |-- id: integer (nullable = true)
    |-- site: string (nullable = true)
    |-- categorie: string (nullable = true)
    |-- site deliste: boolean (nullable = true)
    |-- en peril: boolean (nullable = false)
    |-- pays: string (nullable = true)
    |-- url: string (nullable = true)
    ```
    > **Remarque :**<br> Les types `long` et `integer` représentent tous deux des entiers, la différence majeure réside dans le nombre de bytes. Avec `integer`, on est en 4-bytes et les valeurs peuvent aller de -2,147,483,648 à 2,147,483,647. En `long` on est sur 8-bytes et les valeurs vont de -9,223,372,036,854,775,808 à 9,223,372,036,854,775,807.<br>
    Pour notre cas, le type `integer` est suffisant.
