# Transformation de la donnée
Fabric propose différents outils adéquats pour réaliser vos transformations de données.

<img src="https://github.com/Nicholas0777/Fabric-Workshops/blob/main/Module%20Complete%20Data%20Pipeline/images/Fabric/Fabric-tools-usage-guide.png" alt="" width="80%"/> <br>
[Référence > Microsoft : Implémenter une architecture en médaillon dans Fabric](https://learn.microsoft.com/fr-fr/training/modules/describe-medallion-architecture/3-implement-medallion-archecture-fabric)

Dans la suite du travail, nous allons voir comment réaliser les transformations de données à travers différents outils.<br>
En faisant fie des règles de bonnes pratiques (le but de la démo est d'explorer les fonctionnalités de Fabric), selon son profil (expérience, compétences) et le besoin, le *développeur* Fabric peut passer par :
- **Les dataflow Gen2 :** <br>
Les développeurs Power BI se tourneront naturellement vers cette solution au vu de la forte similarité de l'interface avec dataflow classique (+ Power Query)
- **Les notebooks :** <br>
Similaire à ce que l'on peut faire avec des solutions comme Databricks sur Azure ou les instances Spark sur Amazon EMR.<br>
Les profils data scientists ou data engineers pourront éxécuter des scripts Python ou Spark (PySpark, SQL, R ou Scala) pour effectuer des transformations de données.
- **Les pipelines de données :** <br>
Elles offrent une automatisation des actions de traitement de la de donnée, comparable à des solutions ETL comme Azure Data Factory, Google Cloud Dataflow, AWS Glue, etc.

> **Note :**<br>Il est possible éventuellement de combiner différents outils pour effectuer ces traitements (Par exemple, Pipeline + Dataflow Gen2)


## Route
