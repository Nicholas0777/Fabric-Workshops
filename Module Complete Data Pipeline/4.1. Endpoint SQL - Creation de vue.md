# Point de terminaison SQL : Création d'une vue
Pour des besoins de BI (et pour les exercices suivants), nous allons créer une vue `view_sites_unesco_full` à l'aide des tables existantes dans notre **Lakehouse Silver**.

# 1. Quelques définitions 
## 1.1. Qu'est-ce qu'une vue ?
Une vue est une table virtuelle dans une base de données SQL. Elle est définie par une requête SQL qui sélectionne des données d'une ou plusieurs tables sous-jacentes. Contrairement à une table, une vue ne contient pas de données elles-mêmes; elle affiche simplement les données stockées dans d'autres tables.

### Quelques avantages d'utiliser une vue :
- **Simplification des Requêtes Complexes :** Les vues permettent de masquer la complexité des requêtes SQL. Une requête complexe peut être encapsulée dans une vue, permettant aux utilisateurs de la requêter comme s'il s'agissait d'une table simple.
- **Sécurité et Permissions :** Les vues peuvent être utilisées pour restreindre l'accès aux données sensibles. Vous pouvez créer une vue qui n'inclut que les colonnes ou les lignes autorisées pour certains utilisateurs, puis leur accorder l'accès à cette vue uniquement.
- **Réutilisation de Requêtes :** Les vues permettent de réutiliser des requêtes SQL définies une fois pour être utilisées plusieurs fois par différents utilisateurs ou applications. Cela augmente la consistance et réduit le risque d'erreurs.

# 2. [Pratique] Création d'une vue depuis le point de terminaison SQL
1. Ouvrir le point terminaison SQL du **Lakehouse Silver**. Puis dans Home, cliquer sur **New SQL Query** afin de créer une nouvelle fenêtre dans laquelle le code sera écrit.
    <img src="https://github.com/Expertime/exp-data-fabric-workshops/blob/main/Images/SQL-Endpoint/SQL-Endpoint-create-new-query.png" alt="" width="100%"/>

2. Copier le code ci-dessous et coller dans la nouvelle fenêtre puis exécuter le code
    ```sql
    SELECT 
        whsd.[date_dinscription] AS 'annee_inscription',
        whs.[id],
        whs.[site],
        whs.[categorie],
        whs.[site_deliste],
        whs.[en_peril],
        ROUND( whsd.[superficie_en_ha], 2) AS 'superficie',
        ROUND( whsd.[zone_tampon_en_ha], 2) AS 'zone_tampon',
        whsd.[latitude],
        whsd.[longitude],
        p.[pays] AS 'pays',
        p.[code_pays],
        rg.[nom_francais] AS 'pays_ref_geo',
        rg.[continent],
        rg.[sous_continent],
        rg.[sous_continent_v2],
        p.[region],
        p.[code_region],
        whs.[url]
    FROM [DE_LH_Silver].[dbo].[ref_geo] AS rg
    RIGHT JOIN [DE_LH_Silver].[dbo].[pays] AS p
        ON p.[code_pays] = rg.[iso2]
    INNER JOIN [DE_LH_Silver].[dbo].[world_heritage_sites] AS whs
        ON whs.[pays] = p.[pays]
    INNER JOIN [DE_LH_Silver].[dbo].[world_heritage_sites_details] AS whsd
        ON whsd.[id] = whs.[id]
    ```
    L'output de la requête contient **1.303** observations. 
    > **Question :** Combien d'observations obtient-on en réalisant un `INNER JOIN` entre `ref_geo` et `pays` ? Pourquoi cette différence de résultats ? <br>
3. Pour sauvegarder le résultat en tant que vue, sélectionner l'ensemble de la requête puis cliquer sur **Save as view**<br>

### [Optionnel] Modifier la vue existante
Il peut arriver que pour diverses raisons, on ait envie de modifier une vue existante (ajouter ou supprimer des colonnes, modifier le type de jointure, etc.). Pour cela, on utilise la commande `CREATE OR ALTER`
```sql
CREATE OR ALTER VIEW [dbo].[view_sites_unesco_full] AS (SELECT 
    whsd.[date_dinscription] AS 'annee_inscription',
    whs.[id],
    whs.[site],
    whs.[categorie],
    whs.[site_deliste],
    whs.[en_peril],
    ROUND( whsd.[superficie_en_ha], 2) AS 'superficie',
    ROUND( whsd.[zone_tampon_en_ha], 2) AS 'zone_tampon',
    whsd.[latitude],
    whsd.[longitude],
    p.[pays] AS 'pays',
    p.[code_pays],
    rg.[nom_francais] AS 'pays_ref_geo',
    rg.[continent],
    rg.[sous_continent],
    rg.[sous_continent_v2],
    p.[region],
    p.[code_region],
    whs.[url]
FROM [DE_LH_Silver].[dbo].[ref_geo] AS rg
RIGHT JOIN [DE_LH_Silver].[dbo].[pays] AS p
    ON p.[code_pays] = rg.[iso2]
INNER JOIN [DE_LH_Silver].[dbo].[world_heritage_sites] AS whs
    ON whs.[pays] = p.[pays]
INNER JOIN [DE_LH_Silver].[dbo].[world_heritage_sites_details] AS whsd
    ON whsd.[id] = whs.[id]
)
GO
```

### Réponse à la question
 Avec le `INNER JOIN` on passe de **1303** à **1302** observations. Cela signifie qu'une valeur de la clé de jointure dans `pays` n'est pas présente dans `ref_geo`. Pour vérifier cette valeur on va faire une jointure externe droite (`RIGHT JOIN`) et garder uniquement les valeurs nulles dans la clé de jointure de `ref_geo` (jointure excluante).

```sql
SELECT 
    whs.[id],
    whs.[site],
    p.[pays],
    p.[code_pays],
    rg.[nom_francais] AS 'pays_ref_geo',
    rg.[continent],
    rg.[sous_continent],
    rg.[sous_continent_v2],
    p.[region],
    p.[code_region]
FROM [DE_LH_Silver].[dbo].[ref_geo] AS rg
RIGHT JOIN [DE_LH_Silver].[dbo].[pays] AS p 
    ON p.[code_pays] = rg.[iso2]
INNER JOIN [DE_LH_Silver].[dbo].[world_heritage_sites] AS whs
    ON whs.[pays] = p.[pays]
WHERE rg.[iso2] IS NULL
```
Le site ID **148** ne possède pas de **code_pays**, raison pour laquelle si on veut garder cet élément dans la vue, on réalise une jointure externe entre `pays` et `ref_geo`.
