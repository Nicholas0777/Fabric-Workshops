# 2. Extraire les données liées aux surfaces
Dans cette section, nous allons créer une fonction Python qui extrait les chiffres et les séparateurs décimaux d'une chaîne de caractères, puis nous allons l'appliquer à un DataFrame PySpark pour nettoyer et transformer les données.
1. Nous commençons par définir une fonction `extract_digits` qui prend en entrée une chaîne de caractères et retourne une nouvelle chaîne ne contenant que les chiffres et les séparateurs décimaux, en remplaçant le séparateur décimal par un point :
    ``` python
    def extract_digits(input_str,decimal_sepator=","):
        result = ''
        if type(input_str)==str:
            for char in input_str:
                if char.isdigit() or char == decimal_sepator :
                    result += char
            result = result.replace(decimal_sepator,'.')
        else :
            result = None
        return result

    print( extract_digits("7 778 988,4") )
    ```
    _Output :_
    ```
    7778988.4
    ```

2. Ensuite, nous allons définir une **UDF** (User Defined Function) PySpark pour appliquer notre fonction `extract_digits` aux colonnes du DataFrame.
    ``` python 
    from pyspark.sql.functions import udf, col, when#, lit
    import pyspark.sql.types as T

    # Définir l'UDF PySpark à partir de la fonction extract_digits
    udf_extract_digits = udf( extract_digits, T.StringType() )

    # Appliquer l'UDF aux colonnes 'Bien' et 'Zone tampon', puis les convertir en float
    df = df.withColumn( 'Bien', udf_extract_digits(col('Bien')).cast("float") )\
                .withColumn( 'Zone tampon', udf_extract_digits(col('Zone tampon')).cast("float") )
                
    # Afficher les 5 premières lignes du DataFrame résultant
    display( df.limit(5) )
    ```

# 3. Conversion des coordonnées géographiques du format DMS au format DD
## 3.1. Explication
Les coordonnées DMS sont largement utilisées dans la cartographie, la navigation et d'autres applications géospatiales où une précision détaillée de la position est essentielle. Ils sont composés de trois parties distinctes : **Degrés (°) - Minutes (') - Secondes ('')**.<br>
*Par exemple, une coordonnée DMS comme 40° 26' 46.8" N indique : 40 degrés de latitude nord, 26 minutes, 46.8 secondes.*<br>

Dans la carthographie numérique ou les systèmes de géolocalisation, les coordonnées en **DD (Degrés - Décimaux)** sont préférés aux coordonnées DMS parce qu'ils sont plus simple d'utilisation, précis et interopérables. <br>

Pour des besoins de visualisations dans Power BI, nous allons devoir utiliser les coordonnées en **DD** et donc au préalable convertir les coordonnées géographiques au format **DMS** que nous avons actuellement dans notre base de données en coordonnées au format **DD**.<br><br>
### Formule de conversion
$$ 
    DD = \text{Degrés} + \left(\frac{\text{Minutes}}{60}\right) + \left(\frac{\text{Secondes}}{3600}\right)
$$
Exemple :
$$ 
    DMS \ (40^\circ\ 26'\ 46.8''\ N) \rightarrow DD = 40 + \frac{26}{60} + \frac{46.8}{3600} = 40.44633
$$

> Si le point est situé au Sud *(S)* ou à l'Ouest *(W)*, les coordonnées seront exprimées avec une valeur négative.

## 3.2. [Pratique] Réaliser la manipulation sur un dataframe Spark
On peut réaliser la conversion sur le DataFrame de plusieurs manières, par exemple en utilisant des fonctions lambda sur Pandas, des méthodes de transformation de colonnes, ou des UDF avec PySpark (cf. **Note**). <br>
Dans un premier temps, nous allons créer une fonction pour séparer la latitude de la longitude, et une autre pour effectuer le calcul présenté plus haut.
1. Créer une fonction pour séparer la latitude de la longitude `separate_DMS_latitude_from_longitude`
    ```python
    def separate_DMS_latitude_from_longitude(input_str):
        """
            Séparer une chaine de caractères contenant les coordonnées en dms de la latitude et longitude en deux

            Args:
                x (str): Coordonnées à séparer en deux (2).
                Format [Degres Minutes Secondes Point_Cardinal_Latitude Degres Minutes Secondes Point_Cardinal_Longitude]

            Returns:
                result (list) : Liste de deux (2) chaînes de caractères, 
                où le premier élément est la latitude et le deuxième la longitude.

            Example:
            >>> separate_DMS_latitude_from_longitude('0 41 23.496 N 90 30 4.748 W')
            ['0 41 23.496 N', '90 30 4.748 W']
        """
        #result = None
        if type(input_str)==str:
            input_str_split = input_str.split(" ")
            result = [" ".join(input_str_split[:4]), " ".join(input_str_split[4:])]
        else :
            result = None
        
        return result
    init_coords = '0 41 23.496 N 90 30 4.748 W'
    print( f"Coordonnées \"{init_coords}\" deviennent {separate_DMS_latitude_from_longitude(init_coords)}")
    ```
    _Output :_
    ```
    Coordonnées "0 41 23.496 N 90 30 4.748 W" deviennent ['0 41 23.496 N', '90 30 4.748 W']
    ```
2. Créer une fonction de conversion `dms_to_dd`
    ```python
    def dms_to_dd(dms_str):
        # Separer dms_str par l'espace
        parts = dms_str.split() if dms_str!=None else [0,0,0,'']
        
        # Extraire les degrees, minutes, secondes et la direction
        degrees = float(parts[0])
        minutes = float(parts[1])
        seconds = float(parts[2])
        direction = parts[3]
        
        # Convertir coordonnees DMS en decimal degrees
        dd = degrees + minutes / 60 + seconds / 3600
        
        # Coordonnees negatives si Sud ou Ouest
        if direction in ['S', 'W']:
            dd *= -1
        
        return round(dd,4)
    lat = '0 41 23.496 N'
    print( f"{lat} (en DMS) devient {dms_to_dd(lat)} (en DD)")
    ```
    _Output :_
    ```
    0 41 23.496 N (en DMS) devient 0.6899 (en DD)
    ```
2. Appliquer les fonctions `dms_to_dd` au dataframe PySpark. On effectuera un peu plus de manipulations (en comparaison aux dataframes Pandas où il faut juste utiliser la méthode `apply`). En passant par les UDF, il est nécessaire de préciser le format de l'output (Chaîne de caractères --> StringType(), Nombre à virgule flottante --> DoubleType(), etc.)<br>
    ``` python
    from pyspark.sql.functions import udf, col, when
    import pyspark.sql.types as T

    # Définir l'UDF pour séparer les coordonnées DMS en latitude et longitude
    udf_separate_DMS_latitude_from_longitude = udf( separate_DMS_latitude_from_longitude, T.ArrayType(T.StringType()) )
    # Définir l'UDF pour convertir les coordonnées DMS en degrés décimaux (DD)
    udf_convert_dms_to_dd = udf( dms_to_dd, T.DoubleType() )

    # Appliquer les UDF au DataFrame
    tmp_df = df.withColumn( 'new', udf_separate_DMS_latitude_from_longitude( col('Coordonnées_en_DMS') ) )\
            .withColumn('Latitude', when(col('new').isNotNull(), udf_convert_dms_to_dd(col('new')[0])) )\
            .withColumn('Longitude', when(col('new').isNotNull(), udf_convert_dms_to_dd(col('new')[1])) )\
            .drop('new')

    display(tmp_df.limit(5))
    ```

> **Note:** <br>
    Les fonctions **lambda** et les **UDF** (User Defined Functions) diffèrent principalement en termes d'utilisation et de performance.<br> 
    Les fonctions **lambda** sont des fonctions anonymes, idéales pour des transformations simples et rapides sur de petits ensembles de données, généralement utilisées avec Pandas. <br>
    Les **UDF**, en revanche, sont définies par l'utilisateur pour des transformations complexes sur de grands ensembles de données dans PySpark. Elles permettent de traiter des données distribuées, mais peuvent être moins performantes que les fonctions natives de Spark en raison de la sérialisation des données (*processus de conversion d'un objet en un format qui peut être facilement stocké ou transmis, et est nécessaire pour passer les données entre Python et le moteur Spark*).

## 5.1. Conversion du type des colonnes
On l'a vu dans la requête `df_reorder.printSchema()`, le type de donnée de certaines n'est pas le bon. Avant de créer la table delta dans le Lakehouse, il faudrait affecter le bon type de données à la colonne en question. Pour cela, on utilise `withColumn`, `col` et `cast`.
``` python 
spark_df = df_reorder.withColumn("id",col("id").cast(IntegerType())) \
                .withColumn("dossier",col("dossier").cast(StringType())) \
                .withColumn("site",col("site").cast(StringType())) \
                .withColumn("date dinscription",col("date dinscription").cast(IntegerType())) \
                .withColumn("latitude",col("latitude").cast(DoubleType())) \
                .withColumn("longitude",col("longitude").cast(DoubleType())) \
                .withColumn("superficie en ha",col("superficie en ha").cast(DoubleType())) \

spark_df.printSchema()
```
