# Exploration du Point de terminaison SQL (SQL Analytics Endpoint)
On l'a vu précédemment, lors de la création d'un Lakehouse, deux artéfacts sous-jacents sont créés : un modèle sémantique et un SQL analytics endpoint. Dans ce document, nous nous concentrerons exclusivement sur le SQL analytics endpoint. Nous allons donc :
- Explorer quelques fonctionnalités de cet artifact
- Créer des vues SQL

# Présentation du SQL analytics endpoint
Le **SQL analytics endpoint** est un outil de data warehousing qui permet aux utilisateurs de se connecter aux données stockées dans un Lakehouse à l'aide de SQL ou différents outils basés sur SQL. 
Cet endpoint améliore l'analyse de données basée sur SQL en offrant flexibilité, facilité d'utilisation et intégration avec d'autres outils existants.
### Quelques avantages du SQL analytics endpoint
- **Accès direct à SQL :** Cela rend l'outil accessible pour utilisateurs qui sont déjà familiarisés avec le langage SQL.
- **Exploration des données et création de rapports :** Les utilisateurs peuvent facilement explorer leurs données, exécuter des requêtes complexes, générer des rapports et visualiser leurs résultats sans avoir à déplacer les données entre les outils.
- **Intégration avec d'autres outils :**  L'endpoint prend en charge les connexions à partir de divers outils et applications basés sur SQL, tels que SQL Server Management Studio (SSMS), Power BI, dbt, etc ([MS Fabric - Data Warehouse > Connectivity to data warehousing in Microsoft Fabric](https://learn.microsoft.com/en-us/fabric/data-warehouse/connectivity)).

# Synapse Data Warehouse VS le SQL Analytics Endpoint
Ces deux outils de warehousing qui semblent assez similaires, n'adressent toutefois pas les mêmes cas d'usage.<br>
Le **Synapse Data Warehouse** permet d'exécuter un large éventail d'opérations SQL (comme l'ajout, la modification ou suppression de données) et garantit que ces changements sont réalisés de manière sécurisée et fiable.<br>
En revanche, le **SQL Analytics Endpoint** est une version "light" du **Synapse Data Warehouse** qui, généré automatiquement est en lecture seule permettant d'interroger et d'analyser les données sans les manipuler directement.

## Tableau récapitulatif des différences
> _Tableau inspiré du tableau récapitulatif que l'on peut retrouver dans la documentation officielle Microsoft :  [What is data warehousing in Microsoft Fabric?](https://learn.microsoft.com/en-us/fabric/data-warehouse/data-warehousing#compare-different-warehousing-capabilities)_

| **Capacités / Fonctionnalités**            | **Synapse Data Warehouse**                  | **SQL Analytics Endpoint du Lakehouse**      |
|---------------------------------------------|----------------------------------------------|---------------------------------------------|
| **Utilisation principale**     | Gestion complète des données, transformation, requêtes et transactions                  | Requêtes et analyses sur les tables Delta du Lakehouse         |
| **Support des Transactions T-SQL**           | Oui                                          | Non                                         |
| **Types de Requêtes T-SQL Supportées**       | DQL, DML, DDL*                                | DQL, DDL limitée à la création de vues et fonctions TVF inline |
| **Accès en Lecture/Écriture**                | Lecture et écriture                           | Lecture seulement                            |
| **Création de tables**         | Contrôle total de la création, chargement et transformation des tables                    | Tables générées automatiquement à partir du Lakehouse         |
| **Data ingestion**      |  SQL, pipelines, dataflows                                       | Spark, pipelines, dataflows, shortcuts                                                      |
| **Bilan**                             | Haute performance pour les requêtes transactionnelles complexes. | Limité aux actions de lecture et à la création de vues et fonctions TVF (Table Valued Function) inline. Ne prend pas en charge les modifications directes sur les données. |

> **Notes** : On peut diviser le langage SQL en cinq sous-langages :
>- **DCL (Data Control Language):** pour administrer les bases de données.<br>
    **Principales commandes :** `GRANT`, `REVOKE`
>- **DDL (Data Definition Language):** pour définir et modifier la structure des objets de base de données dans un système de gestion de base de données relationnelle.<br>
    **Principales commandes :** `CREATE`, `DROP`, `ALTER`, `TRUNCATE`, `RENAME`
>- **DML (Data Manipulation Language) :** pour manipuler les données stockées dans une base de données, telles que l'insertion, la mise à jour, la suppression et la récupération des données.<br>
    **Principales commandes :** `INSERT`, `UPDATE`, `DELETE`, ...
>- **DQL (Data Query Language) :** pour interroger et récupérer des données à partir d'une base de données relationnelle.<br>
    **Principale commande :** `SELECT`
>- **TCL (Transaction Control Language):** pour gérer les transactionse.<br>
    **Principales commandes :** `COMMIT`, `ROLLBACK`, `SET TRANSACTION`, ...

| **dbo**                | Schéma par défaut pour les objets créés par les utilisateurs comme les tables et les vues. ## Exemples de requêtes avec les vues des autres schémas
- `INFORMATION_SCHEMA` <br> Lister les colonnes de toutes les tables du schéma `dbo`
    ```sql
    SELECT TABLE_CATALOG, TABLE_NAME, COLUMN_NAME, COLUMN_DEFAULT
    FROM [INFORMATION_SCHEMA].[COLUMNS]
    --FROM [DE_LH_Silver].[INFORMATION_SCHEMA].[COLUMNS]
    WHERE TABLE_SCHEMA = 'dbo';
    ```
-  `queryinsights` <br>Identifier les requêtes que vous avez exécutées au cours des 30 dernières minutes (cf. https://learn.microsoft.com/fr-fr/fabric/data-warehouse/query-insights#examples)
    ```sql
    SELECT * 
    FROM queryinsights.exec_requests_history 
    WHERE start_time >= DATEADD(MINUTE, -30, GETUTCDATE()) 
        AND login_name = USER_NAME();
    ```
- `sys` <br>Identifier les requêtes en cours qui prennent du temps à s'éxécuter (cf. https://learn.microsoft.com/en-us/fabric/data-warehouse/monitor-using-dmv):
    ```sql
    SELECT request_id, session_id, start_time, total_elapsed_time
    FROM sys.dm_exec_requests
    WHERE status = 'running'
    ORDER BY total_elapsed_time DESC;
    ```
