3. Aller dans **New** > **Notebook**. Après quelques secondes, un notebook s'ouvrira automatiquement et vous serez sur l'interface ci-dessous.<br>

## 1.3. Présentation des bibliothèques que nous allons utiliser
Pour la suite du travail, nous allons utiliser les bibliothèques suivantes :
### Pandas
- **pandas**<br>
    Elle est utilisée pour le traitement de données. Avec elles sont fournies des structures de données comme les DataFrames pour travailler efficacement avec des données tabulaires.
- **numpy**<br>
    Elle fournit des fonctions et des structures de données pour effectuer des calculs numériques efficaces, notamment avec des tableaux et des matrices. Elle est souvent utilisée en conjonction avec Pandas pour des opérations de manipulation de données plus complexes.
- **json**<br>
    Elle permet de sérialiser des objets Python en chaînes JSON et de désérialiser des chaînes JSON en objets Python. Elle est essentielle pour travailler avec des fichiers de données au format JSON.
- **os** <br>
    Elle permet d'interagir avec le système d'exploitation. Elle fournit des fonctions pour la manipulation des chemins de fichiers, l'exploration des répertoires, et bien plus encore.

### PySpark
- **pyspark.sql.functions**<br>
    Elle contient un ensemble de fonctions permettant de manipuler les colonnes des DataFrames PySpark. On peut trouver des fonctions pour des agrégations, des transformations, des filtres, etc.
    | Fonction      | Description                                                                                 |
    |---------------|---------------------------------------------------------------------------------------------|
    | col           | Pour référencer une colonne dans un DataFrame.                                              |
    | when          | Pour des conditions logiques similaires à _if-else_.                                          |
    | udf           | Pour définir des fonctions utilisateurs afin d'appliquer des transformations personnalisées aux colonnes. |
    | lit           | Pour créer des colonnes littérales avec une valeur constante.                               |
    | regexp_replace| Pour effectuer des remplacements basés sur des expressions régulières.                      |
    | split         | Pour diviser une chaîne en plusieurs parties selon un délimiteur.                           |
    | concat        | Pour concaténer plusieurs colonnes ensemble.                                                |
    | avg, sum, min | Pour les calculs d'agrégation.                                                              |
- **pyspark.sql.types**<br>
    Elle contient les types de données nécessaires pour définir le schéma des DataFrames PySpark. On l'utilise pour formater les colonnes des DataFrames avec les types de données appropriés.
    | Type          | Description                                                                                       |
    |---------------|---------------------------------------------------------------------------------------------------|
    | StringType    | Pour les colonnes contenant des chaînes de caractères.                                            |
    | IntegerType   | Pour les colonnes contenant des entiers.                                                          |
    | FloatType     | Pour les colonnes contenant des nombres à virgule flottante.                                      |
    | DoubleType    | Pour les colonnes contenant des nombres à double précision.                                       |
    | StructType    | Pour définir des schémas de DataFrames avec des structures complexes.                             |
    | StructField   | Utilisé avec `StructType` pour définir les champs du schéma.                                      |
    | ArrayType     | Pour les colonnes contenant des listes d'éléments.                                                |


> **Bon à savoir :**<br>
    La *bibliothèque* désigne ensemble de modules regroupés pour effectuer des tâches spécifiques. Par exemple, la bibliothèque standard de Python contient une variété de modules pour des fonctions comme la manipulation de fichiers, le calcul mathématique, la gestion des dates, etc.<br>
    Contrairement à la *bibliothèque*, le *module* lui, est un fichier contenant du code réutilisable tel que des fonctions et des variables, facilitant ainsi l'organisation logique du code et sa réutilisation dans différents programmes.<br><br>
    La communauté de développeurs Python crée continuellement des bibliothèques et modules pour améliorer les pratiques de codage. Ces bibliothèques couvrent divers domaines tels que la data science (`SciPy`, `Scikit-learn`, `PyTorch`) et la dataviz (`Matplotlib`, `Seaborn`, `Geopandas`). Importer ces bibliothèques permet d'accéder à ces outils spécialisés, améliorant ainsi l'efficacité. 
    <br>
    On peut trouver ces projets depuis [PyPI](https://pypi.org/) (Python Package Index), le principal référentiel en ligne pour les packages Python.

## 1.4. Différence entre Pandas et PySpark
La différence principale entre `Pandas` et `PySpark` réside dans leur architecture et leur capacité de traitement : Pandas est optimal pour les analyses sur une seule machine avec des ensembles de données modérés, offrant une prise en main facile et des performances rapides. En revanche, PySpark est spécialement conçu pour le traitement distribué sur des clusters, ce qui le rend adapté aux environnements Big Data avec des volumes de données massifs et nécessitant un parallélisme élevé.

| **Caractéristique**     | **Pandas**                            | **PySpark**                           |
|-------------------------|---------------------------------------|--------------------------------------|
| **Architecture**        | Utilisation sur une seule machine      | Calcul distribué sur un cluster      |
| **Scalabilité**         | Convient pour des données jusqu'à plusieurs gigaoctets en mémoire | Gère des volumes de données allant de gigaoctets à pétaoctets en distribué |
| **Performance**         | Optimisé pour le traitement sur une machine unique | Performances élevées sur des clusters avec parallélisme distribué |
| **Traitement des données** | Manipulation de données en mémoire, rapide pour les opérations sur des ensembles de données modérés | Adapté pour le traitement massivement parallèle de grandes quantités de données |
| **Facilité d'utilisation** | Facile à prendre en main pour les utilisateurs familiarisés avec Python et les dataframes | Requiert une courbe d'apprentissage plus élevée en raison de la configuration du cluster et des concepts de traitement distribué |
| **Déploiement**         | Installation simple sur une machine locale | Nécessite une configuration et un déploiement sur un cluster Spark |
| **Écosystème**          | Vaste écosystème de bibliothèques et d'outils pour la manipulation, l'analyse et la visualisation des données. Comprend des packages comme NumPy, SciPy, Matplotlib, Seaborn pour des tâches spécifiques en data science et dataviz. | Écosystème intégré avec des outils pour le Big Data et le machine learning sur Spark, incluant Spark SQL, MLlib pour le machine learning, GraphX pour le traitement de graphes, et des intégrations avec d'autres technologies comme Hadoop, Kafka, et les services cloud. |
| **Cas d'utilisation**   | Idéal pour les analyses ad-hoc, les prototypes et les petits ensembles de données | Adapté aux environnements Big Data avec des volumes de données massifs et des besoins de traitement distribué |


> **Note :** <br>Étant donné que nos fichiers actuels ne sont pas de grande volumétrie, nous pouvons utiliser `Pandas` sans crainte de problème de performance. <br> 
Cependant, si vous souhaitez vous familiariser avec l'environnement Spark, `PySpark` peut être une bonne option.<br>
